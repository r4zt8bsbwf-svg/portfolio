ShiftOpt

Constraint-based workforce scheduling — reduced manual schedule generation from ~2 hours to under 5 minutes.

Role Sole developer
Stack C++ · OOP · Constraint-based assignment
Type Operations / algorithms
Status In use

The problem

Scheduling 75 employees by hand is the kind of task that takes hours, produces inconsistent results, and quietly creates staffing gaps in high-priority positions. ShiftOpt replaces that process with a constraint-based assignment engine written in C++.

Manual: ~2 hours per schedule. ShiftOpt: under 5 minutes. Same constraints, fewer conflicts.

What it optimizes for

  • Employee availability and time windows
  • Position qualifications and skill levels
  • Minimum staffing requirements per shift
  • Position priority rankings (critical roles first)
  • Consistency — keep employees in the same role rather than rotating unnecessarily

Position-based logic

Different roles need different staffing levels at different times. IPOS might need 2 workers continuously; drinks only needs 1; drive-thru staffing extends later than front counter. The scheduler supports variable staffing counts per role with time-dependent requirements, so a single run produces a schedule that respects every constraint at once.

Architecture

  • Object-oriented C++ — employee classes, availability maps, position priority systems, shift blocks
  • Constraint evaluation functions that score candidate assignments
  • Dynamic scheduling arrays/vectors for the working set

The algorithm fills high-priority positions first, then iteratively assigns remaining staff while minimizing future bottlenecks. After each placement it recalculates coverage gaps and remaining available labor before moving on.

Where it's going

The current version is functional but improvements I'm working toward: a menu-driven management UI, editable employee profiles persisted to disk, support for the full ~75-employee database, easier per-location position customization, and automated conflict resolution when constraints are over-specified.

What it taught me

Real-world scheduling isn't a clean optimization problem — it's a stack of soft preferences and hard constraints that contradict each other. The interesting work is in deciding which to relax and in what order. C++ was the right call: predictable performance, full control over data structures, and the codebase stays readable as constraints accumulate.
